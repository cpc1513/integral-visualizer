# 积分视界实施计划

## 阶段 1：工程与视觉骨架

- 建立 React + Vite + TypeScript 工程、HashRouter、测试环境和 GitHub Pages 构建配置。
- 抽取确认稿中的颜色、字体、圆角、间距和组件状态为 CSS tokens。
- 完成 `AppShell`、双按钮顶栏、默认“在线计算”路由和响应式双栏布局。

验证：类型检查、生产构建、桌面/手机首屏截图。

## 阶段 2：文档数据

- 编写 `scripts/extract_questions.py`，按文档顺序遍历段落、Word OMML 公式和题目分段。
- 把公式转换为结构化 LaTeX 片段，生成 `src/data/questions.generated.json`。
- 校验题目总数、学年分组、空题面、公式数量和原文备注。

验证：提取脚本可重复运行；生成数据正好 85 题；抽查首、中、末年份公式。

## 阶段 3：计算台

- 定义 `IntegralSpec` 与五类示例。
- 实现 MathLive 公式编辑、符号键盘、类型选择和动态边界表单。
- 实现公式与结构化参数同步、字段校验和本地恢复。

验证：组件测试覆盖类型切换、符号插入、参数更新和错误提示。

## 阶段 4：计算内核

- 在模块 Worker 中按需加载 Pyodide 与 SymPy。
- 实现普通、重、曲线、曲面积分的符号求值与步骤摘要。
- 符号失败时再加载 SciPy/mpmath 数值回退。
- 主线程实现加载状态、计算超时、Worker 重建和结果缓存。

验证：五类种子示例各至少一个结果正确；超时不会冻结 UI。

## 阶段 5：可视化

- Plotly 按需加载，避免阻塞首屏。
- 实现普通定积分填充、二重积分区域、三重积分体积、参数曲线与参数曲面。
- 加入三维旋转、缩放、重置视角和文本摘要。

验证：五类种子场景可绘制；非法/无界区域显示错误而不是误导图形。

## 阶段 6：往年真题

- 实现年份/知识点/类型筛选、文本搜索、紧凑题目列表和解答折叠。
- 用 KaTeX 渲染题目与解答公式；转换失败时保留原始文字并显示来源备注。
- 为可映射题目提供“载入在线计算”。

验证：85 题均可浏览，筛选结果正确，列表采用 `content-visibility` 控制长页开销。

## 阶段 7：交付验收

- 运行测试、类型检查和生产构建。
- 在应用内浏览器检查默认路由、两个导航、一次完整计算、题目筛选和三维旋转。
- 检查桌面与手机视口；对照确认稿完成文案、布局、色彩、字体、边框、密度和状态审计。
- 修复所有可见偏差，清理临时 QA 文件后提交。
