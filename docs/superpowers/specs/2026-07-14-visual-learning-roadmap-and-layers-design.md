# 积分可视化教学路线图与图层控制设计

## 产品目标

网站继续保持“面向中国大学生高数复习的一体化积分区域教学工具”定位。借鉴 Desmos、GeoGebra、CalcPlot3D、Maple 和 Wolfram 的交互思想，但不扩展成通用 CAS 或独立绘图命令平台。

核心目标不仅是给出积分答案，还要让学生看懂积分区域、边界、方向、积分次序和相关定理之间的关系。

## 分阶段路线图

### 第一阶段：图层控制

统一管理二维 Plotly 与三维 Three.js 图中的积分区域、边界、表面、方向箭头、坐标轴和网格。该阶段为后续截面、投影、向量场和定理联动建立稳定协议。

### 第二阶段：截面和积分次序动画

加入 `x=c`、`y=c`、`z=c` 截面滑块，以及二重、三重积分的内层线段、中层截面和外层整体扫描动画。

### 第三阶段：格林、高斯、斯托克斯专题模式

- 格林：平面区域、边界曲线、方向箭头、向量场和面积分/曲线积分对应。
- 高斯：封闭曲面、内部区域、外法向量、通量和散度体积分对应。
- 斯托克斯：空间边界曲线、可选跨界曲面、曲线方向、曲面法向和右手定则联动。

### 第四阶段：坐标变换演示

并列展示直角坐标与极坐标、柱坐标或球坐标区域，解释坐标网格和雅可比因子的来源。

### 第五阶段：参数与输出

增加参数滑块、图片导出和可复现的分享链接。账号、云存储、完整 CAS 命令语言和社区功能不在当前路线内。

## 第一阶段详细设计

### 交互入口

在可视化面板顶部、“重置视角”旁增加“图层”按钮。点击后打开紧凑弹出面板，不占用左右主体布局。

面板只显示当前绘图实际支持的图层：

- 普通积分：函数曲线、积分填充、坐标网格、坐标轴。
- 二重积分：积分区域、边界、坐标网格、坐标轴。
- 三重积分：积分区域或上/下表面、坐标网格、坐标轴。
- 曲线积分：曲线、方向箭头、坐标网格、坐标轴。
- 曲面积分：曲面、边界（存在时）、坐标网格、坐标轴。

切换题型或公式生成新的绘图规格时使用该图形的默认图层状态，避免沿用旧图的隐藏状态导致误判。所有默认值保持当前页面外观不变。

如果所有主体图层均被关闭，绘图区保留坐标环境，并显示轻量提示“主体图层已隐藏”，不将其作为绘图错误。

### 统一图层协议

`plotSpec` 为每张图声明可用图层：

- 稳定的图层键；
- 中文标签；
- 默认可见性；
- 是否属于主体图层；
- 对应 Plotly trace 或 Three.js group。

首批稳定图层键包括：

- `integrand`：函数曲线或被积对象；
- `region`：二维填充或三维积分区域；
- `boundary`：边界线；
- `upperSurface` / `lowerSurface`：累次积分上下表面；
- `surface`：曲面积分曲面；
- `curve`：曲线积分路径；
- `arrows`：方向箭头；
- `grid`：坐标网格；
- `axes`：坐标轴与标签。

后续的 `projection`、`slice`、`vectorField` 和 `normalVectors` 使用同一协议扩展，不修改面板架构。

### Plotly 路径

绘图规格记录每个 trace 所属图层。切换曲线、填充、边界和表面时使用 Plotly trace visibility；切换网格和坐标轴时使用 layout 更新。

图层操作只更新已有图，不重新解析公式，不重新构造标量场，也不触发 Python/SymPy 计算。

### Three.js 路径

`ThreeIsosurfaceCanvas` 和 `ThreeCurveCanvas` 将场景对象放入独立 `Group`：

- 主体几何组；
- 边界或方向箭头组；
- 网格组；
- 坐标轴与标签组。

图层状态变化只修改 `group.visible` 并重绘，不重新创建 WebGLRenderer、几何体、材质或 OrbitControls。公式变化和组件卸载仍按现有流程释放全部 GPU 资源。

### 状态与组件

新增独立 `LayerControl` 组件负责弹出面板、图层开关和键盘可访问性。`VisualizationCanvas` 持有当前图层状态，并将其分别传给 Plotly 或 Three.js 渲染器。

图层状态不写入积分公式和题库数据，也不触发计算结果失效。首版不持久化到 localStorage，避免不同图形之间出现难以解释的隐藏状态。

### 响应式与可访问性

- 桌面端面板从顶部按钮向下展开。
- 窄屏面板保持在绘图区内部并允许换行，不遮挡主要输入区。
- 按钮使用 `aria-expanded`、`aria-controls`；开关支持键盘操作。
- 点击面板外部或按 Escape 关闭面板。

## 错误与降级

- Three.js 初始化失败并回退 Plotly 时，重新采用回退规格支持的图层集合。
- 某图层没有对应对象时不显示无效开关。
- Plotly 更新失败时保留当前图，不把公式或积分结果标为失败。
- 主体全部隐藏只显示提示，不触发错误状态。

## 验证标准

- 所有现有绘图默认外观与交互保持不变。
- 二维区域的填充、边界、网格和坐标轴可以分别开关。
- 三维条件区域可单独开关主体、网格和坐标轴。
- 斯托克斯交线可单独开关管线与方向箭头，切换时 WebGL 上下文不重建。
- 累次三重积分的上、下表面可分别开关。
- 图层操作不调用 Python/SymPy，不重新生成标量场或曲线路径。
- 切换题型后恢复合理默认图层。
- 桌面和窄屏布局无溢出、遮挡或不可点击项。
- 单元测试、TypeScript 检查、生产构建和本地页面交互检查通过。
