# 数学输入框、自定义积分区域与计算生命周期设计

日期：2026-07-10

## 目标

解决计算台的三个直接问题：参数与边界仍以普通文本显示 LaTeX 源码；共用符号键盘只能向顶部完整公式输入；复杂 Python 计算在 20 秒后被前端强制终止。同时为二重积分和三重积分增加由用户直接输入不等式约束的自定义积分区域。

本轮不增加收藏、账号、云同步或新的顶部栏目。导航仍为“在线计算 / 往年真题”。

## 统一数学输入

所有数学内容统一使用 MathLive `MathfieldElement`：

- 顶部完整积分公式。
- 被积函数。
- 积分变量与参数变量。
- 积分上下限。
- 曲线参数方程与曲面参数方程。
- 向量场分量。
- 自定义积分区域约束和扫描范围。

内部继续保存标准 LaTeX 字符串。例如数据中的 `\sqrt{2}\sin u\cos v` 在界面中显示为排版后的数学公式，不显示反斜杠源码。

新增可复用的 `MathInputField` 组件。它负责创建、同步和销毁 MathLive 实例，维持标签、紧凑尺寸、禁用状态和可访问名称。原 `ExpressionField` 不再承载数学数据。

## 共用符号键盘与焦点路由

计算台维护唯一的 `activeMathfield`。所有数学输入框在获得焦点或被点击时注册为当前目标；失焦不立即清空目标，使用户可以先点输入框，再点下方符号键盘。

符号键盘始终向 `activeMathfield` 的当前光标或选区插入内容。顶部完整公式框不再拥有特殊优先级。当前目标使用现有蓝色焦点边框，并通过键盘区域的可访问说明标识目标字段。

当字段卸载或积分类型切换时，如果它仍是当前目标，焦点管理器选择新页面中的第一个可编辑数学框。不存在目标时符号按钮禁用。

## 自定义条件区域

二重积分和三重积分在“参数与边界”中增加模式切换：

- `累次上下界`：保留现有内层、中层、外层边界模型，适合精确符号积分。
- `条件区域`：用户逐行输入 LaTeX 等式或不等式，所有行取交集。

支持的关系包括 `=、<、≤、>、≥`，支持 `x、y、z`、常数、四则运算、幂、根式、绝对值和已支持的初等函数。例如：

- `x^2+y^2\le 4`
- `x\ge 0`
- `y\ge x^2`
- `z\le 4-x^2-y^2`

条件区域提供增删约束按钮，并提供必需的扫描范围：二重积分为 `x/y` 最小值和最大值，三重积分增加 `z`。扫描范围也是 MathLive 输入框。示例切换到条件区域时，以当前累次上下界可推导出的范围作为初值；无法推导时使用 `[-5,5]`。

`MultipleIntegralSpec` 增加区域模式和条件区域对象。累次边界继续保留，使用户可以在两种模式之间切换而不丢失输入。

## 条件区域可视化

约束解析器把关系两侧转换成可计算表达式，并生成“满足/不满足”判定函数。

- 二维区域在扫描范围中规则采样，用遮罩填充满足全部条件的区域，并绘制边界等值线。
- 三维区域构造约束违反度标量场，使用 Plotly `isosurface` 或等价网格显示交集边界与内部透明体。
- 等式采用与扫描范围成比例的数值容差；不等式使用严格的符号方向。
- 无有效点、表达式无法编译或扫描范围非法时，不沿用旧图，画布显示具体错误与对应约束。

条件模式下顶部完整公式使用区域记号 `\iint_D` 或 `\iiint_\Omega`，而不是伪造不存在的累次上下限。

## 条件区域计算

累次上下界模式继续使用现有 SymPy → SciPy 流程。条件区域无法转换为可靠的累次上下界时，不把指示函数交给 `nquad` 长时间自适应搜索，而是使用 NumPy/SciPy Sobol 准蒙特卡洛采样：

1. 在用户扫描范围内生成低差异样本。
2. 过滤满足全部约束的点。
3. 计算被积函数均值乘以扫描盒体积。
4. 返回数值结果、采样点数量和统计误差估计。

该结果明确标记为“条件区域数值积分”，不伪装成精确值。空区域直接报错。

## Python 计算生命周期

删除进入 `computing` 状态后 20 秒自动 `terminate()` 的逻辑。数学计算不会因为固定时间自动停止。

计算期间：

- 主按钮显示“停止计算”，点击后才终止 Worker。
- 结果区实时显示从请求开始计算的耗时。
- 符号计算持续 12 秒后出现“转为数值计算”。点击后终止当前阻塞 Worker，启动数值优先的新任务；Pyodide 下载文件由浏览器缓存复用。
- 用户停止后保留上一次已完成结果，状态显示“已由用户停止”。
- 新计算开始前取消仍在运行的旧请求，避免结果串题。

网络运行时加载仍保留独立的 180 秒超时，只用于 CDN 或包下载失败，错误文案明确为网络加载问题。它不作为数学计算时限。

题库可视化规格增加 `preferredComputeMode: exact | numeric`。普通定积分和简单累次积分默认精确；复杂曲线、曲面、条件区域和已知容易卡住的题默认数值优先。用户仍可手动选择另一种方法。

## 组件与数据流

- `MathInputField`：单个可视化 LaTeX 输入。
- `MathfieldFocusContext`：注册并保存当前 MathLive 目标。
- `SymbolKeyboard`：读取当前目标并插入模板。
- `RegionModeEditor`：切换累次上下界与条件区域。
- `ConstraintRegionEditor`：维护约束列表、变量和扫描范围。
- `constraintExpression`：解析关系、编译判定器并提供错误位置。
- `plotSpec`：增加二维遮罩和三维隐式区域分支。
- `computeClient`：返回可取消任务，管理停止和数值重试。
- `pythonWorker`：支持 `exact`、`numeric` 和 `constraint-numeric` 请求模式。

## 错误处理

- 单个数学框含无效 LaTeX 时保留用户输入，在该框和画布显示错误，不回退到旧区域。
- 约束缺少关系符、包含未知变量或范围上下限颠倒时禁止计算，但允许继续编辑。
- 手动停止、网络失败、符号失败、数值失败使用不同状态和文案。
- Worker 被停止后，下次计算自动创建干净 Worker，不要求刷新页面。

## 测试与验收

- 数学输入组件测试：外部 LaTeX 同步、用户编辑回传、字段注册和卸载。
- 焦点路由测试：依次点击主公式、上下限、参数方程后，符号只进入当前字段。
- 约束解析测试：圆、三角形、抛物线夹域、球和抛物面立体。
- 绘图测试：二维/三维条件区域产生有限采样，空区域和非法范围正确报错。
- 计算生命周期测试：无 20 秒硬终止、手动停止、12 秒数值切换、旧请求不会覆盖新请求。
- 浏览器测试：桌面和手机端编辑多个数学框，自定义圆盘与球体区域，验证图形、计算状态和无横向溢出。
- 回归测试：37 道严格真题仍能载入、原累次边界模式、Python 精确计算和 GitHub Pages 构建保持可用。
