# 累次上下界三重积分闭合实体设计

## 背景与问题

当前 `triplePlot()` 仅根据内层变量的上下界生成顶部和底部两个 Plotly `surface`。以单位圆柱为例，这只能显示 `z=0` 与 `z=1` 两个圆盘；Plotly 不会自动推断投影区域边界处的侧壁，因此图形不是闭合实体。

## 目标

- 累次上下界三重积分显示为不透明、闭合、可旋转的实体外壳。
- “实心”指视觉上完整的实体边界，不引入内部体素或剖面采样。
- 保留连续参数曲面的平滑观感，避免重新使用低分辨率等值面提取。
- 非有限值与退化边界不得产生错误薄片或跨定义域连面。

## 方案比较

### 方案 A：补齐参数化边界面（采用）

保留顶部和底部表面，并生成中层上下界侧面及外层上下界端面。各侧面使用参数 `t∈[0,1]` 在内层下界与上界之间插值。

优点是改动集中、数学边界准确、继续使用现有 Plotly 参数曲面；不需要新增渲染器。缺点是边界由多个 surface trace 组成，需统一材质并跳过退化面以避免接缝。

### 方案 B：Three.js 闭合三角网格

将全部边界采样并统一三角化为 `BufferGeometry`。视觉控制更强，但需要新增数据协议、渲染组件和回退路径，超出本次缺陷修复范围。

### 方案 C：有符号场等值面

将全部上下界转换为一个约束违反度标量场，再由 Marching Cubes 提取零等值面。它天然闭合，但会重新引入网格分辨率导致的锯齿，不符合当前平滑优先目标。

## 采用设计

`triplePlot()` 最多生成六类边界：

1. `inner-upper`：内层变量上界；
2. `inner-lower`：内层变量下界；
3. `middle-upper`：中层变量上界处的侧壁；
4. `middle-lower`：中层变量下界处的侧壁；
5. `outer-upper`：外层变量上界处的端面；
6. `outer-lower`：外层变量下界处的端面。

每个 trace 在 `meta.boundaryKind` 中记录稳定类型，测试不依赖数组位置。侧壁的竖直方向由 `t` 在对应点的内层上下界之间插值。所有面使用相同的不透明配色、无等高线材质；边缘通过足够密集的采样保持平滑。

若一个边界面的有效三维跨度或面积低于容差，则视为退化面并跳过。例如单位圆柱在 `x=±1` 处的端面退化为线，不应生成薄片。任何非有限采样继续写为 `null`，不得跨过定义域缺口连接。

条件区域三重积分继续使用现有隐式实体路径，本次仅修改“累次上下界”模式。

## 验证标准

- 单位圆柱包含上下圆盘及两个半圆柱侧壁，从任意斜视角看均为闭合实体。
- 单位立方体生成六个非退化边界面。
- 当前默认抛物面示例不出现多余薄片，退化边界被正确跳过。
- 依赖 `x,y` 的内层上下界在侧壁上使用完整作用域计算。
- 单元测试、生产构建和五类积分端到端测试全部通过。
- 桌面与移动端均无新增溢出、控制台错误或明显交互卡顿。

## 非目标

- 不渲染内部粒子、体素密度或切片。
- 不为 Plotly 增加新的图表类型依赖。
- 不修改积分计算、题库数据或条件区域语义。
