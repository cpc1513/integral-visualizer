# 二维与三维约束边界平滑设计

## 目标

消除条件区域和隐式曲面因布尔采样产生的台阶、尖齿和锯齿观感。在不改变积分区域数学含义的前提下，让二维边界和三维曲面片边缘尽可能连续、平滑。

## 问题根因

- 二维条件区域目前把每个采样点压成 `0 / 1`，Plotly 接收到的是方格掩码。线条平滑只能修饰已经呈台阶状的数据，无法恢复真实边界。
- 三维隐式曲面目前按三角形中心点查询布尔 `mask`，整片保留或删除。跨越约束边界的三角形会组成一圈尖齿。
- 单纯提高采样分辨率只能缩小锯齿，不能消除这种离散裁剪误差。

## 方案

### 统一的连续约束值

约束解析器除判断点是否有效外，还为每条可连续裁剪的不等式输出带符号残差：

- 残差小于等于零表示满足约束；
- 残差大于零表示超出约束；
- 残差等于零就是精确边界。

扫描范围、公式解析和积分计算保持不变。等式仍用于定义主曲线或主曲面，不作为需要封口的区域边界。

### 二维条件区域

二维区域不再把标量场转换成二值掩码。绘图规格直接使用“所有约束残差的最大值”形成连续标量场，并以零等值线作为区域边界。

Plotly 继续负责坐标轴、缩放和悬停；区域内部使用负残差范围填充，零等值线使用连续插值及平滑线条。采样精度适度提高，但主要改善来自连续插值，而不是暴力堆分辨率。

预期效果包括：抛物线、圆、椭圆及多约束交界处不再出现明显像素台阶。数学上本应有尖角的交界仍保留尖角，不做错误的圆角化。

### 三维隐式曲面片

Three.js 标量场规格用每条连续约束的残差数组替代最终布尔 `mask`。Marching Cubes 仍生成由等式定义的主曲面，然后对每个三角形执行连续裁剪：

1. 在三角形顶点处对约束残差做三线性采样。
2. 对跨越零边界的三角形，沿边按残差比例求交点。
3. 使用半空间多边形裁剪依次应用所有不等式。
4. 将裁剪后的多边形重新三角化，并插值顶点法线。

因此圆柱 `x^2+y^2=1` 在 `z` 上下界裁剪后只保留侧面，上下边缘是连续圆环，不额外生成底面。一般曲面片同样保持开放边界，避免把不属于积分对象的封口画出来。

对于无法构造连续残差的非标准条件，保留兼容降级路径，但不让它影响常见的 `≤、≥、<、>` 条件。

## 代码边界

- `plotSpec.ts`：生成二维连续违反量和三维逐约束裁剪场。
- 新的纯几何工具模块：负责三线性采样、三角形半空间裁剪和扇形三角化，不依赖 React 或 DOM。
- `ThreeIsosurfaceCanvas.tsx`：调用几何工具构造最终 BufferGeometry，并继续负责材质、光照、相机和资源释放。
- `VisualizationCanvas.tsx` 的渲染路由和现有 Plotly 回退保持不变。

## 性能

- Three.js 继续按需动态加载。
- 裁剪只处理 Marching Cubes 实际生成的三角形，不扫描额外的稠密三维单元。
- 连续残差数组复用现有采样循环生成，避免再次解析和计算公式。
- 不为了隐藏锯齿盲目提高到过大的分辨率；在当前 48³ 基础上通过几何插值获得更高的视觉精度。

## 验证标准

- 圆柱侧面经上下界裁剪后，上下边缘无周期性三角尖齿且不出现底面。
- 裁剪后顶点都满足约束边界容差，法线为有限非零值。
- 二维圆、抛物线区域边界不再显示方格台阶。
- 球体等三重积分闭合区域保持光滑、实体外观。
- 旋转、缩放、重置视角和 Plotly 降级不回归。
- 全部单元测试、TypeScript 检查和生产构建通过。
