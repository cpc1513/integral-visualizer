# Three.js 隐式积分区域渲染设计

## 目标

将目前由 Plotly `isosurface` 绘制的三维条件区域和复杂隐式曲面切换为 Three.js 渲染，使球体、椭球体、圆锥和其他隐式边界获得封闭、无孔且具有平滑光照的外观，同时保留开放的三元不等式输入能力。

## 范围

本次只替换以下两类图形：

- 三重积分在“条件区域”模式下生成的闭合三维区域。
- 无法转换成显式 `surface` 网格的复杂隐式曲面。

二维区域、普通函数、累次积分上下界、参数曲面、显式曲面和隐式曲线继续使用 Plotly。积分解析、Python/SymPy 计算和题库数据不变。

## 架构

### 标量场

现有约束解析器继续计算每个采样点相对积分区域的约束违反量。所有约束违反量的最大值构成标量场：小于等于零表示区域内部，零等值面表示区域边界。

标量场包含：

- 三个变量名及扫描范围；
- 规则三维采样网格；
- 每个采样点的标量值；
- 建议的等值面阈值零。

### 渲染路由

`VisualizationCanvas` 根据绘图规格选择渲染器：

- 普通 Plotly 规格进入现有 Plotly 渲染流程。
- Three.js 等值面规格进入新的 `ThreeIsosurfaceCanvas`。

Three.js 及其附加组件动态导入，仅在需要三维等值面时加载，避免增加二维页面的首屏执行成本。

### Three.js 场景

`ThreeIsosurfaceCanvas` 负责：

- 使用 Three.js 官方 Marching Cubes 生成闭合网格；
- 使用连续梯度法线和 `MeshStandardMaterial` 获得平滑光照；
- 开启 WebGL 抗锯齿；
- 提供透视相机、轨道旋转、滚轮缩放和重置视角；
- 绘制简洁的三维网格、坐标轴和 `x / y / z` 标签；
- 在容器尺寸变化时自动调整画布和相机。

采样和网格生成使用中高精度，优先保证教学图形的光滑程度。后续如出现复杂区域性能问题，再把标量场计算移动到 Web Worker；本次不重复改造现有计算 worker。

## 生命周期和回退

- 公式或范围改变时，旧场景、控制器、材质、几何体和 WebGL 资源必须释放。
- 组件卸载时停止动画循环，避免后台持续占用 GPU。
- WebGL 或 Three.js 初始化失败时显示明确错误，并允许保留现有 Plotly 等值面作为降级结果。
- 新绘图生成期间继续显示当前加载状态，不阻塞公式编辑。

## 视觉标准

- 闭合区域表面没有规则三角孔洞。
- 默认视角下球面等常见区域不出现明显多面体棱角。
- 曲面具有柔和高光和明暗过渡，但不过度反光。
- 旋转、缩放和重置视角行为与当前网站一致。
- 坐标轴和背景颜色继续匹配现有蓝白视觉风格。

## 验证

- 单元测试验证三重积分条件区域和复杂隐式曲面被路由到 Three.js 标量场规格。
- 单元测试验证标量场包含内部值和边界两侧值。
- 全部现有测试继续通过。
- TypeScript 和生产构建通过。
- 浏览器检查球体 `x^2+y^2+z^2\le1` 的封闭、平滑效果，以及旋转和重置视角。
- 检查页面无框架错误遮罩和相关控制台错误。
