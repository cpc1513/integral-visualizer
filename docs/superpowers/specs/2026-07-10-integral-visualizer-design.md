# 积分视界：网站设计规格

日期：2026-07-10  
状态：视觉方向已确认  
视觉基准：`docs/design/integral-visualizer-concept.png`

## 1. 产品目标

“积分视界”是面向大学高等数学复习的开放式积分题目与可视化工具。网站不设置账号、刷题进度、随机练习、评分或排行榜。它只做两件事：

1. 让用户像使用科学计算器一样，通过积分类型模板、数学符号键盘和可编辑 LaTeX 公式输入积分，并立即看到计算结果、步骤摘要以及二维或可旋转三维积分区域。
2. 公开陈列《高等数学下_积分真题汇编 (1).docx》中的历年题目、原解答和来源说明，并允许把适合可视化的题目载入在线计算页。

网站以 GitHub Pages 为目标部署环境，因此最终产物必须是纯静态前端，不依赖服务器、数据库或私有 API。

## 2. 信息架构

顶部导航只保留品牌“积分视界”和两个主按钮：

- “在线计算”：默认激活，网站打开后首先显示。
- “往年真题”：进入公开题目浏览页。

不添加首页、教程、登录、搜索图标或其他顶级导航。页面切换使用前端路由并兼容 GitHub Pages 刷新与直达。

### 在线计算页

桌面端采用左右双区工作台：左侧约 38% 宽度负责输入和结果，右侧约 62% 宽度是主要可视化画布。首屏无需纵向滚动即可完成一次输入、计算和查看。

左侧自上而下为：

1. 积分类型选择：普通积分、二重积分、三重积分、曲线积分、曲面积分。
2. 完整公式输入：MathLive 数学输入框，支持直接键盘编辑 LaTeX。
3. 数学符号键盘：积分号、根号、常数、变量、三角函数、上下标、分式、集合与不等式等常用符号。
4. 参数与边界：随积分类型改变的结构化字段。结构化字段与完整 LaTeX 双向同步，保证解析稳定。
5. “计算并可视化”主按钮。
6. 计算结果：精确值、数值近似、运行状态、可展开的计算步骤。

右侧为可视化画布：

- 标题固定为“积分区域”。
- 二维场景支持缩放、平移、悬停坐标、重置视角。
- 三维场景支持鼠标拖动旋转、滚轮缩放、平移和重置视角。
- 边界、投影区域、积分方向、曲线路径、曲面法向等信息使用一致的颜色语义和图例。
- 默认示例使用确认稿中的三重积分：由 `z=0` 与 `z=4-x^2-y^2` 围成的区域。

### 往年真题页

该页是开放式资料浏览页，不出现“开始练习”“正确率”“提交答案”等练习系统概念。

- 顶部提供普通文本搜索以及年份、知识点和积分类型筛选。
- 主体使用紧凑的题目列表，不使用大面积卡片瀑布流。
- 每道题显示年份、原题编号、分值、知识点、题目 LaTeX、原解答和来源备注。
- 解答默认折叠，用户可展开查看。
- 适合当前可视化模型的题目显示“载入在线计算”操作；无法可靠转换的题目仍完整展示，但不伪造可视化参数。
- 文档中“原题暂缺”“需重新核对”等说明按原文保留，作为来源备注，不在网站中擅自改写成确定结论。

## 3. 输入与计算模型

### 混合输入

用户可以直接编辑完整 LaTeX，同时通过积分类型模板和参数字段构造公式。内部不依赖纯字符串猜测，而维护统一的 `IntegralSpec`：

- 类型：ordinary、double、triple、line、surface。
- 被积函数与变量。
- 普通/重积分的逐层上下限。
- 曲线积分的参数方程、参数范围、方向和第一/第二类类型。
- 曲面积分的参数化或显式曲面、参数域、方向和第一/第二类类型。
- 原始 LaTeX 与规范化表达式。

MathLive 负责数学编辑体验；自定义同步器把常见 LaTeX 映射为结构化积分字段，mathjs 负责可视化表达式采样。结构化字段是计算与可视化的权威边界数据；完整 LaTeX 始终对用户可见并随之同步。

### 浏览器内 Python

首次点击计算时按需加载 Pyodide，随后在独立 Web Worker 中运行 SymPy 和 SciPy：

1. 优先用 SymPy 求符号精确值。
2. 无闭式结果、符号计算失败或达到时限时，改用 SciPy/mpmath 数值积分。
3. UI 明确区分“精确值”“数值近似”和“未能收敛”。
4. 计算超时通过终止并重建 Worker 处理，避免冻结主线程。
5. Python 运行时只在用户第一次计算时加载；页面其余部分可以立即使用。

不向用户承诺任意复杂积分必然存在闭式解。对于超出首版参数化能力的公式，保留计算结果或错误说明，同时提示用户补充结构化边界。

## 4. 可视化规则

可视化使用按需加载的 Plotly WebGL/Canvas 图层，并由独立的图形规格生成器把 `IntegralSpec` 转换成图形数据。

- 普通定积分：绘制函数曲线、上下限和曲线与基准线之间的有符号填充区域。
- 普通不定积分：绘制原函数代表曲线，并说明常数族；不虚构有限积分区域。
- 二重积分：在二维坐标系中填充投影域并绘制边界；可切换查看被积函数的三维曲面。
- 三重积分：绘制上、下边界曲面及侧边界，使用透明材质表现体积；支持旋转视角。
- 曲线积分：绘制二维或三维参数曲线、起终点和方向箭头；第二类曲线积分同时表现向量场采样。
- 曲面积分：绘制曲面网格、参数边界、选定法向与必要的投影域；第二类曲面积分明确方向。

绘图采样具有上限并在低性能设备上自动降级。表达式不连续、区域无界、边界相交失败或采样包含非实数时，不输出误导图形，而显示具体错误和可修改字段。

## 5. 题目数据

源文档中的题目在构建阶段转换为静态 TypeScript/JSON 数据。每道题使用稳定 ID，并保存：

- 学年、题号、分值、知识点、积分类型。
- 题目段落与 LaTeX 公式。
- 解答段落与 LaTeX 公式。
- 原文备注和数据质量标记。
- 可选的 `IntegralSpec` 预设。

Word OMML 公式先转换为 MathML，再转换为 LaTeX；转换脚本保留在仓库中以便日后重新生成。网站运行时只读取静态数据，不读取 DOCX，也不使用数据库。

浏览器 `localStorage` 只保存用户最后一次计算页输入和界面偏好；不记录身份、成绩或学习行为。

## 6. 视觉系统

实现必须忠实复现确认稿：

- 背景：纯白 `#FFFFFF`；可视化画布使用冷调浅灰 `#F8FAFC`。
- 主文字：深海军蓝 `#102A4C`；次级文字 `#5B6B82`。
- 主色：钴蓝 `#2563EB`；选中和焦点使用同一蓝色语义。
- 成功结果：克制的绿色 `#229A5B`，仅用于有效结果和运行成功。
- 边框：`#DCE4EF`，1px；不使用暖灰或米色替代白底。
- 圆角：输入与小控件 10px，主要面板 14px。
- 阴影：近乎扁平，只使用低透明度的单层阴影。
- UI 字体：`Inter, "Noto Sans SC", "Microsoft YaHei", sans-serif`。
- 数学字体：`"STIX Two Math", "Cambria Math", serif`。
- 间距遵循 8px 基线；按钮、标签和输入框都显式设置字号、行高和字重。

不添加渐变、霓虹光、营销横幅、统计卡、装饰徽章、无意义图标或嵌套卡片网格。

## 7. 组件边界

React + Vite + TypeScript 项目按以下职责拆分：

- `AppShell`：全局页框和顶栏。
- `TopNavigation`：两个主导航按钮和激活状态。
- `CalculatorPage`：组合计算工作台，不承担解析或绘图细节。
- `IntegralTypeSelector`：五种积分类型。
- `FormulaEditor`：MathLive 输入与 LaTeX 同步。
- `SymbolKeyboard`：可访问的数学符号按钮。
- `BoundsEditor`：按积分类型渲染结构化参数。
- `ComputePanel`：Worker 生命周期、状态、结果和步骤。
- `VisualizationCanvas`：Plotly 场景与交互控制。
- `PastExamsPage`：筛选、列表和选题加载。
- `QuestionRow` / `QuestionDetail`：题目与解答展示。
- `integralSpec`：统一数据模型和验证。
- `pythonWorker`：Pyodide、SymPy、SciPy 执行桥接。
- `plotSpec`：把积分规格转换为二维或三维图形数据。

`App` 只负责路由和顶层组合，不写成单一巨型组件。

## 8. 状态、错误与可访问性

- Python 未加载、正在加载、计算中、符号成功、数值回退、超时和错误均有明确状态。
- 表达式与边界校验在计算前执行，错误定位到具体字段。
- Worker 错误不会清空用户输入；重建后可以重试。
- 键盘按钮均为真实 `button`，有名称、焦点环和键盘操作。
- 导航、表单、结果和图形使用语义化区域；图形提供文本摘要，不能只靠颜色传递信息。
- 尊重 `prefers-reduced-motion`。
- 桌面端保持左右工作台；窄于 900px 时依次堆叠输入、结果和画布；手机端画布保持可操作高度，顶部两个导航按钮不溢出。

## 9. 验收标准

### 功能

- 默认进入“在线计算”，两个顶级导航均可访问。
- 五类积分模板、公式键盘、LaTeX 编辑和结构化参数同步工作。
- 至少各有一个普通、二重、三重、曲线和曲面积分示例可成功计算与绘图。
- Python 运行时按需加载，计算不阻塞主线程，并有数值回退与超时恢复。
- 文档中的题目可以按年份、知识点和类型浏览，题目与解答公式正确渲染。
- 可视化支持二维交互和三维旋转、缩放、重置视角。

### 质量

- TypeScript、构建、单元测试和核心交互测试通过。
- 桌面与手机视口无横向溢出、裁切或不可读控件。
- 浏览器实际截图与确认稿在导航文案、布局比例、色彩、字体层级、控件密度、画布主体和结果样式上保持一致。
- 首屏不得出现确认稿之外的营销文案、额外导航或装饰组件。

## 10. 明确不做

- 登录、账号、云同步、数据库。
- 刷题、随机出题、评分、正确率、排行榜。
- 教师后台或在线编辑题库。
- 对所有自由形式数学文字进行通用语义理解。
- 承诺所有积分都能得到闭式解。
- 在首版中托管 Python 服务器。
