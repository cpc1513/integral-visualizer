# 斯托克斯交线光滑管线设计

## 目标

把曲线积分“隐式条件”模式下的稀疏三维点云替换为连续、光滑、可旋转的空间曲线。斯托克斯公式题中的两曲面交线应能清楚展示闭合性、空间位置和积分方向。

## 问题根因

当前实现扫描固定的 `45³` 网格，只保留同时落入两个等式容差带的采样点，再用 Plotly `scatter3d` 绘制。结果不是有序曲线，而是密度随网格和曲面夹角变化的点云；提高采样密度只能增加计算量，无法从根本上消除条带、空洞和断裂。

## 推荐表现

- 默认使用蓝色 Three.js 管线显示曲线，不再显示粒子点云。
- 管径随扫描范围缩放，约为有效视野跨度的 `0.6%–0.9%`。
- 闭合曲线首尾连续；开放曲线保留真实端点。
- 沿曲线放置少量方向箭头，箭头不会遮挡主体。
- 曲线积分编辑区增加“正向 / 反向”控制。反向只改变曲线顺序和箭头，不改变几何形状。
- 题库题目没有可靠结构化方向数据时，不根据中文题干猜测方向；使用默认追踪方向，并允许用户按题意一键反转。避免展示一个看似精确但实际错误的方向。

## 曲线求解

### 参数曲线

参数方程模式继续按参数区间采样，但输出有序三维点列。点列经过按弧长重采样后直接生成管线，参数正反向决定箭头方向。

### 两隐式曲面的交线

设两个主等式写成 `f(x,y,z)=0` 和 `g(x,y,z)=0`。使用数值延拓追踪交线：

1. 在用户扫描范围内做低分辨率种子搜索，并用 Newton 校正把候选点投影到两曲面的交线上。
2. 在交线点计算数值梯度，切线方向取 `∇f × ∇g`。
3. 沿切线做预测步，再联立 `f=0`、`g=0` 和垂直校正平面做 Newton 校正。
4. 自适应调整步长：曲率大时缩小，平缓处放大。
5. 当新点回到起点邻域且切线方向连续时闭合曲线；碰到扫描范围或不等式边界时形成开放端点。
6. 对剩余种子重复追踪并去重，以支持多个互不相连的曲线分支。

追踪点需要满足等式残差容差和额外不等式约束。梯度近平行、Newton 不收敛或曲线退化时停止当前分支，不把错误点连接进管线。

## 数据与组件

`plotSpec.ts` 输出独立的 Three.js 曲线规格，包含：

- 一条或多条有序点列；
- 每条路径是否闭合；
- 扫描范围和坐标轴名称；
- 当前方向；
- 管径建议值。

新增纯数值模块负责梯度、Newton 校正、延拓、闭合检测和分支去重。该模块不依赖 React、DOM 或 Three.js，便于单元测试。

新增 `ThreeCurveCanvas` 负责：

- 使用 `CatmullRomCurve3` 和 `TubeGeometry` 生成光滑管线；
- 用小型圆锥体沿切线放置方向箭头；
- 复用现有 Three.js 蓝白场景、相机、轨道控制和重置视角行为；
- 在公式或方向改变时释放几何体、材质、控制器和 WebGL 资源。

`VisualizationCanvas` 按规格选择 Plotly、Three.js 曲面或 Three.js 曲线。Three.js 仍按需动态加载。

## 精度与性能

- 种子搜索只用于找到交线入口，不再把所有命中网格点直接绘制。
- 延拓目标点数控制在每条闭合曲线约 `240–600` 点，曲率决定实际数量。
- 每个点都经过方程校正，视觉密度与扫描网格无关。
- 计算超过安全迭代上限时返回明确错误，并保留现有 Plotly 点云作为兼容回退，但不作为正常展示。

## 验证标准

- 球面 `x²+y²+z²=1` 与平面 `x+y+z=1` 的交线显示为单条闭合光滑圆管，没有散点或粗条带。
- 柱面 `x²+y²=1` 与平面 `y+z=2` 的交线连续闭合。
- 球面与抛物面的交线能够闭合，点均满足两个方程的残差容差。
- 正向 / 反向切换后几何不变，点列和箭头方向反转。
- 参数曲线继续正确显示并使用相同视觉语言。
- 旋转、缩放、重置视角以及 Three.js 失败时的 Plotly 回退不回归。
- 全部单元测试、TypeScript 检查和生产构建通过。
